## Additional out-of-band checks (run-scenarios.sh)

Not part of the compat scoreboard but run alongside:

- Declarative-admission denial (CEL): `spec.color=black` on
  widgets produced HTTP 422 with the configured field-path
  and message — PASS.
- Declarative-admission mutation (JSONPath default): widgets
  with no `spec.title` come back with `spec.title=Untitled` —
  PASS (after a consumer-facing friction point: the JSONPath
  string must NOT have a leading `.`; see 0031 FINDINGS for
  rough-edge detail).
- Stitched Get (0024 metastore): both APIs show `uid`,
  `resourceVersion`, `creationTimestamp` in `-o yaml` that come
  from the host-etcd ResourceMetadata CR, not the backend —
  PASS.
- Cross-API isolation: deleting all widgets + the
  widgets APIDefinition would leave gadgets untouched (not
  exercised destructively; we confirmed both APIServices remain
  Available after cross-API writes) — PASS.
- GC reconciler: after all objects deleted, the widget store's
  ResourceMetadata rows are removed by the next sweep (30s
  interval, 10s grace) — PASS.
- Graceful SIGTERM: `kubectl delete deploy aggexp` causes the
  PreShutdown hook to delete both APIService objects before the
  pod exits; kubectl-side discovery no longer lists the groups —
  PASS.

## Known-gap expectations confirmed (not unexpected breakage)

- `kubectl explain <kind>` fails with "couldn't find resource"
  (not a 404 from the AA, but a client-side schema-not-known
  response — kubectl's explain path reads the per-group v3 OpenAPI
  endpoint, which the library doesn't register for
  post-PrepareRun-installed groups).
- `kubectl apply --server-side` fails at
  `managedfields.NewTypeConverter`: "schema error: no type found
  matching: io.k8s.apimachinery.pkg.apis.meta.v1.ObjectMeta". This
  is the SSA typed-converter gap 0027 and 0030 named; fixing it
  needs library-level apiserver changes, not substrate.

No unexpected regressions; the substrate held.
